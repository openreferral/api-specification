This is a tool for encouraging a conversation around the design of this API. Some who are technical, but others who are not. It uses Github issues to help manage the ongoing conversations around the API and its design. It runs 100% on Github pages, leverage JavaScript and the Github API to do what it does. Users who have been added to the repo, can pass in a personal token using the "token" query string variable, and then will be able to annotate and comment on each endpoint.

All annotations to each endpoint are just added to this Github repos issue management system, using the path as the title, and applying the HTTP verb as a label. You can access everything under issue management, in the Github repository for this project.
